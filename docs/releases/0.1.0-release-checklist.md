# ADTM 0.1.0 Release Checklist

Target release: `0.1.0`

Purpose: track the first public package release as a repo-owned artifact before
or in parallel with a GitHub issue or milestone.

Use [`maintainer-runbook-template.md`](maintainer-runbook-template.md) as the
structure for the private owner/signoff record. Do not commit filled-in owner
handles or credential material here.

## Owners

- [ ] Primary PyPI owner assigned
- [ ] Backup PyPI owner assigned
- [ ] GitHub repository admin coverage confirmed
- [ ] All publishing accounts use 2FA
- [ ] Private maintainer runbook updated with concrete owner handles

## Packaging

- [ ] `pyproject.toml` version confirmed
- [ ] `CHANGELOG.md` updated
- [ ] `python -m pytest -q` passes from package root
- [ ] sdist and wheel build cleanly
- [ ] `twine check dist/*` passes
- [ ] clean wheel install smoke passes
- [ ] clean sdist install smoke passes

## Docs And Policy

- [ ] README reflects current adapter defaults, benchmark policy, and quickstart
- [ ] `SECURITY.md`, `CONTRIBUTING.md`, and `CODE_OF_CONDUCT.md` reviewed
- [ ] benchmark docs carry date, hardware, corpus, and cost context
- [ ] dependency/license notes rechecked for exact release versions
- [ ] PyMuPDF release posture explicitly accepted before publish
- [ ] Private maintainer runbook records the PyMuPDF acceptance decision

## Hygiene

- [ ] secret scan clean
- [ ] no local `.env`, tokens, private corpora, caches, model weights, or generated artifacts tracked
- [ ] `.gitignore` covers local build and benchmark outputs

## Publishing

- [ ] TestPyPI trusted publisher configured
- [ ] PyPI trusted publisher configured
- [ ] protected `pypi` environment requires manual approval
- [ ] test upload to TestPyPI verified from clean virtualenv
- [x] release tag policy decided
- [ ] production upload approved
- [ ] PyPI project page verified after publish
- [ ] GitHub release notes cut from `CHANGELOG.md`

## Status

Current state on `2026-04-27`:

- package docs, issue templates, contribution docs, and publishing guidance are present
- release checklist is tracked in-repo pending first public milestone/issue creation
- `0.1.x` release tag policy is documented: protected annotated `v*` tags, CI-built artifacts only, no mandatory artifact signing for the first public release
- package-owned public benchmark corpus and dated public-fixture benchmark snapshot are present
- a safe maintainer runbook template exists; concrete owner handles still belong in a private copy
- concrete PyPI owner handles still need to be assigned privately
