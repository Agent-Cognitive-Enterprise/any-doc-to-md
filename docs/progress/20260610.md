## Slice 10 Review Fix: Make Default Paragraph Repair Observable

**Status:** complete  
**Date:** 2026-06-10  
**Role:** build after red-team review

**Review finding addressed:**

The first Slice 10 wiring was mechanically correct, but the default
`ParagraphRepairSettings.min_paragraphs=20` meant realistic short row-sliced
documents were rejected before the evidence gates ran. That created a
claim/reality gap: default repair was documented and wired but hard to observe
through the real CLI or public benchmark workflow.

**Implemented changes:**

- Lowered the default paragraph-repair document-size floor from 20 prose blocks
  to 8.
- Kept the semantic gates unchanged: structure ratio, short-fragment ratio,
  no-terminal ratio, lowercase-start ratio, continuation-pair ratio,
  continuation-run length, content preservation, structural-count preservation,
  and quality delta still all gate acceptance.
- Added `examples/benchmark-corpus/row-sliced-note.txt`, a small package-owned
  public fixture that demonstrates the default repair path through the real
  in-house adapter and tournament pipeline.
- Added a narrow `.gitignore` unignore for `examples/benchmark-corpus/` because
  the existing `benchmark-*/` ignore pattern hid new package-owned public
  fixture files.
- Updated the public benchmark reproduction guide for the four-fixture current
  smoke and documented that the older dated public-fixture snapshot remains a
  historical three-document measurement.
- Changed README wording from "current" to "dated" for the 2026-04-27 public
  fixture snapshot.

**Tests added/updated:**

- Detector tests now prove a 13-fragment row-sliced document is detected by
  default and a 7-fragment version remains below the floor.
- Application tests now prove the same realistic 13-fragment document is
  accepted and repaired in memory.
- CLI tests now run a real conversion twice, default `auto` and explicit `off`,
  and assert only the default path publishes joined Markdown plus winner
  paragraph-repair artifacts.
- Public benchmark tests now include `row-sliced-note.txt` and assert the real
  tournament writes `index_fixed.md`, `index_paragraph_repaired.md`, and
  `paragraph_repair_report.json` for that fixture.
