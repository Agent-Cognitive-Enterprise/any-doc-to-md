# Progress 2026-06-12

## Table Fidelity Planning: In-House PyMuPDF Markdown Tables

**Status:** planning only  
**Date:** 2026-06-12  
**Role:** planning

**Objective:**

Prepare the `correct-tables-conversion` branch for a SABRE implementation that
preserves detected PDF tables as native Markdown tables in the fast `inhouse`
path, without making optional heavy converters part of the default path.

**No implementation yet:**

- No source code was changed for table extraction.
- No tests were added yet.
- No CLI behavior, converter behavior, staging shape, or output contract changed.

**Current baseline observations:**

- Current PDF conversion extracts text and images only:
  `pdf_converter.convert()` calls `extract_pdf_blocks()` and
  `assemble_markdown(...)`.
- Current table handling is text-only: `detect_block_kind(...)` marks a text
  block as `table` only if already-extracted text contains pipe characters.
- Current assembler emits `block_kind == "table"` as a fenced code block, not a
  native Markdown table.
- `document.override.yaml` support already exists through `load_overrides(...)`,
  so a per-document opt-out such as `table_extraction: off` fits the current
  architecture.
- Local PyMuPDF has `Page.find_tables()` and `Table.to_markdown(clean=False,
  fill_empty=True)`. A small synthetic grid table PDF was recognized by
  PyMuPDF during planning.

**Planning decisions:**

- Keep `inhouse` as the fast default adapter.
- Use PyMuPDF table APIs already available through the existing dependency.
- Do not add Docling, pdfplumber, Camelot, tabula-py, pandas, or tabulate to the
  default path.
- Prefer generated synthetic PDFs in tests over committed binary fixtures for
  the first slices.
- Add extraction support behind small isolated helpers before changing default
  output.
- Do not ship a default-on slice that knowingly duplicates table text. Either
  keep table extraction internal/off until suppression exists, or combine
  default-on table emission with overlap-based text suppression.
- Suppress PyMuPDF's optional layout-package recommendation before table scans
  if it appears during tests or CLI runs. Keep this suppression narrow to the
  table-scan call site so genuine diagnostics are not hidden globally.
- Scope the first table feature to tables PyMuPDF actually detects. In practice,
  the first synthetic fixtures should use ruled grid tables. Borderless aligned
  tables are out of scope unless a later slice deliberately changes
  `find_tables(...)` strategy and proves it.
- Precedence rule: when a PyMuPDF `TableBlock` overlaps ordinary text blocks,
  the native table wins and overlapped flattened text is suppressed. When
  PyMuPDF detects no table, the existing pipe-text `block_kind == "table"`
  fallback remains unchanged and still emits fenced code until a separate slice
  explicitly changes that legacy behavior.
- Treat default-on native table output as an intentional output change. When the
  default-on slice lands, update `CHANGELOG.md`, README, and relevant docs with
  compatibility framing and the `table_extraction: off` escape hatch.

**Proposed test fixtures and assertions:**

Important fixture rule: these are logical cell values, not literal PDF source
text to insert with pipe characters. Test PDFs should draw table ruling lines
with `page.draw_line(...)` and insert plain cell text into each cell. Do not draw
literal pipe glyphs in the PDF table; PyMuPDF `find_tables()` can fail or emit
garbage columns for pipe-delimited text.

Basic table PDF logical content:

```text
Quarterly Metrics

Table 1. Equipment status by owner.

Component    Status    Owner
Pump A       Stable    Rina
Valve B      Watch     Omar

After the table, this paragraph must survive.
```

Expected assertions:

```python

assert "|---|---|---|" in md
assert "|Pump A|Stable|Rina|" in md
assert "|Valve B|Watch|Omar|" in md
assert "After the table, this paragraph must survive." in md
assert md.count("Pump A") == 1
assert md.count("Valve B") == 1
```

Recommended generator shape:

```python
rows = [
    ["Component", "Status", "Owner"],
    ["Pump A", "Stable", "Rina"],
    ["Valve B", "Watch", "Omar"],
]
```

Draw full internal row and column ruling lines for those cells with
`page.draw_line(...)` and insert only the strings above into their cells. A
single bounding rectangle is not enough: PyMuPDF's default line strategy needs
the interior gridlines to split rows and columns, so an outer rectangle alone
yields zero detected tables.

## Slice 1: Table Block Model And No-Op Assembler Compatibility

**Status:** complete
**Date:** 2026-06-12
**Role:** build

**Objective:**

Add first-class PDF table block plumbing without changing existing conversion
output when no table blocks are supplied.

**Implemented changes:**

- Added `TableBlock` beside `TextBlock` and `ImageBlock` in
  `_pdf_blocks.py`.
- `TableBlock` records page, bbox, Markdown, row/column counts, source,
  warning text, column, and computed `y_mid`.
- Updated `sort_key(...)` so table blocks share the same page/column/y ordering
  as text and image blocks.
- Added optional trailing `table_blocks: list[TableBlock] | None = None` to
  `assemble_markdown(...)`; existing positional callers remain compatible.
- When explicitly supplied, table blocks are sorted into the normal block stream
  and emitted as native Markdown table text, not fenced code.
- Existing extraction and converter wiring remain untouched. No table extraction
  runs by default yet.

**Tests added/updated:**

- Added a focused unit test proving `TableBlock` sorts between text and image
  blocks using the existing PDF block ordering.

**Review fix:**

Red-team review found that the initial Slice 1 plumbing did not lock two claims
with tests and would let the next extraction slice produce two visible defects:
duplicate table text and table-caption teleporting.

- Added assembler-side duplicate suppression when explicit `TableBlock`s are
  supplied. Text blocks whose center is inside a same-page table bbox and whose
  overlap ratio against the text block is at least `0.65` are dropped before
  formatting. Caption-like text blocks are preserved.
- Made caption attachment table-aware. `Table N.` captions attach to the nearest
  same-page `TableBlock`; captions above a table stay before the table and
  captions below a table stay after it. Existing figure/image caption behavior
  remains.
- Added tests proving `table_blocks=[]` is a byte-identical no-op, table blocks
  emit native Markdown rather than fenced code, table captions remain with their
  table across page breaks, and flattened text inside a table bbox is not
  emitted twice.
- Kept extraction and converter wiring untouched. Table extraction still does
  not run by default.
- Retained `row_count`, `col_count`, `source`, and `warning` on `TableBlock` as
  extraction/report evidence fields for Slice 2 and later; the assembler
  intentionally formats only the trusted Markdown payload.

**Follow-up review fix (caption leapfrog):**

A second red-team pass found that making caption attachment relocate a caption
next to its target could reorder body text: a caption separated from its image or
table by an intervening paragraph was pulled out of natural order to hug the
target, moving that paragraph above the caption.

- A caption is now relocated only when it is already adjacent to its target in
  reading order (nothing but other captions lies between them in the sorted
  block stream). Otherwise the caption stays in its natural position. This
  applies uniformly to image and table captions.
- Added a regression test proving a non-adjacent `Table N.` caption keeps natural
  reading order (caption, intervening paragraph, table) instead of leapfrogging.

## Slice 2: PyMuPDF Page Table Extraction Helper

**Status:** complete
**Date:** 2026-06-12
**Role:** build

**Objective:**

Add a focused page-level PyMuPDF table extraction helper that returns
`TableBlock`s and warnings, without integrating it into full PDF conversion yet.

**Implemented changes:**

- Added `src/anydoc2md/format_converters/_pdf_tables.py`.
- Added `extract_page_tables(...)`, which calls `page.find_tables()`, converts
  each detected table with `table.to_markdown(clean=..., fill_empty=...)`, and
  returns `(list[TableBlock], list[str])`.
- Added `looks_like_markdown_table(...)` to validate that generated Markdown has
  a plausible GFM separator row before emitting a `TableBlock`.
- Nonfatal behavior is explicit: missing `find_tables()`, detector exceptions,
  malformed Markdown, undersized tables, and `to_markdown(...)` failures become
  warnings instead of conversion errors.
- The helper suppresses PyMuPDF's optional `pymupdf_layout` recommendation at
  the table-scan call site when the installed PyMuPDF exposes
  `no_recommend_layout()`.
- No `pdf_converter` wiring was added. Default PDF conversion behavior is still
  unchanged.

**Tests added/updated:**

- Added `tests/test_pdf_tables.py`.
- Tests cover Markdown-table validation, missing `find_tables()`, detector
  exceptions, malformed Markdown filtering, fallback for older/no-arg
  `to_markdown()`, and real extraction from a generated ruled-grid PDF.
- The generated PDF fixture draws full internal row and column ruling lines and
  inserts plain cell text, matching the branch planning constraint.

## Slice 2: red-team residual fixes

**Status:** complete
**Date:** 2026-06-12
**Role:** red-team / build

**Findings addressed:**

- Removed the unused `TableBlock.warning` field. Per-table problems are surfaced
  through the warnings list returned by `extract_page_tables(...)`; no code path
  ever emitted a block carrying a warning, so the field was dead and redundant
  with that channel. The `source` provenance field is retained.
- Loosened the brittle exact-bbox assertion in the generated-PDF test from a
  default `pytest.approx` (~1e-6) to `abs=1.0`, so a 1pt change in how PyMuPDF
  derives a table bbox no longer breaks the test for no functional reason.
- Added branch tests that previously had no coverage: undersized-table filter,
  non-int row/column dimensions, `to_markdown` raising directly, the
  `TypeError` -> no-arg `to_markdown()` fallback then also failing, and the
  malformed-bbox -> zero-bbox fallback.

**Deferred (out of scope for Slice 2):**

- Detection `strategy` is not plumbed; borderless/whitespace-aligned tables are
  not detected and produce no warning. This is already an explicit, documented
  scope boundary (see HANDOFF "Notes for next session") to be revisited only
  when a later slice changes and tests `find_tables(...)` strategy.
