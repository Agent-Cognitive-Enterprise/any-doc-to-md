## Slice: Orthogonal Content-Preservation Evidence

**Status:** complete
**Date:** 2026-06-09
**Role:** build

**Objective:**
Fix the residual red-team finding that `content_preserved` is a single document-level boolean: one ambiguous hyphen-wrap taints the whole `RepairDraft`, so a gate that hard-rejects on `content_preserved=False` can never auto-accept a document containing any hyphen join.

**Files touched:**

- `src/anydoc2md/paragraph_repair/model.py`
- `src/anydoc2md/paragraph_repair/repairer.py`
- `tests/test_paragraph_repair_model.py`
- `tests/test_paragraph_repair_repairer.py`

**Implemented changes:**

- Redefined `content_preserved` as a whitespace-insensitive character round-trip (`"".join(text.split())` equality). It is a true loss guard: `True` for character-preserving space and hyphen joins, `False` only on real content loss (e.g. the old hyphen-dropping bug).
- Added `RepairDraft.hyphen_join_count`, counted in `_merge_group`, so the ambiguity of collapsed end-of-block hyphen boundaries is reported orthogonally instead of overloading the loss boolean.
- Documented the `_merge_group` right-edge invariant that makes recomputing the join kind during the fold safe.
- Updated the two hyphen tests to the new semantics (`content_preserved=True`, `hyphen_join_count` reported) and added an orthogonality test proving a clean run stays content-preserved when a hyphen run shares the document.
- Added a direct `_content_preserved` guard test demonstrating it ignores whitespace but catches a dropped character.

## Slice: Add Staging-Directory Paragraph Repair Helper

**Status:** complete
**Date:** 2026-06-09
**Role:** build

**Objective:**
Apply paragraph repair to one adapter staging directory while preserving raw `index.md` and avoiding the shared tournament `index_fixed.md` slot.

**Files touched:**

- `HANDOFF.md`
- `docs/progress/20260609.md`
- `src/anydoc2md/paragraph_repair/application.py`
- `src/anydoc2md/paragraph_repair/__init__.py`
- `tests/test_paragraph_repair_staging_application.py`

**Implemented changes:**

- Added `apply_paragraph_continuity_repair(adapter_name, adapter_staging_dir, source_path, *, settings=None) -> ParagraphRepairReport`.
- The helper reads `adapter_staging_dir/index.md`, runs `repair_markdown_paragraph_continuity(...)`, and writes `index_paragraph_repaired.md` only when the in-memory repair is accepted.
- Raw `index.md` is never modified.
- The helper does not write, overwrite, or delete `index_fixed.md`; that filename remains reserved for the existing tournament fix/publish path.
- Missing staging directories return a no-op report with `attempted=False`, `accepted=False`, and reason `staging_dir_missing`; missing `index.md` returns reason `index_md_missing`.
- Rejected, disabled, and missing-input repairs remove stale `index_paragraph_repaired.md`/`paragraph_repair_report.json` files owned by this helper while preserving unrelated `index_fixed.md`.
- Accepted repairs write `paragraph_repair_report.json` with schema version, creator, adapter name, portable source/staging attribution, raw/repaired artifact SHA-256 fingerprints and byte sizes, `owns_output=True`, `publishes_index_fixed=False`, and the bounded `ParagraphRepairReport.to_dict()` payload.
- Exported the staging helper from the internal `paragraph_repair` package.

**Red-team follow-up:**

- Reworked the helper after review identified `index_fixed.md` as an already-owned tournament artifact consumed by fix application, selection, and publishing.
- Removed the unenforced `owns_index_fixed=True` claim and the write to `index_fixed.md`.
- Added regression coverage proving an accepted paragraph repair does not overwrite a pre-existing `index_fixed.md`.
- Added stale-owned-artifact cleanup for this helper's own output and removed absolute source/staging paths from the sidecar payload.
- Fixed the atomic-write cleanup path so temp files are removed even when the write operation fails before `os.replace(...)`.
- Made disabled staging repair's cleanup behavior explicit: disabled runs clear stale paragraph-repair-owned artifacts but preserve unrelated `index_fixed.md`.
- Future consumers must require both `index_paragraph_repaired.md` and `paragraph_repair_report.json` before trusting a paragraph-repair candidate.
